# Декомпозиція сайту Contrast для Яцківа Дениса Руслановича

## 1. Загальна структура

Сайт Contrast є односторінковим landing page для платформи проведення інтерактивних вебінарів. Сторінка послідовно веде користувача від головної продуктової пропозиції до демонстрації інтерфейсу, соціального доказу, AI-можливостей, функцій залучення аудиторії, мозаїки інструментів і фінального заклику до дії.

```text
Contrast page
├── header
├── main
│   ├── hero
│   ├── social-proof
│   ├── ai-section
│   ├── engagement
│   ├── feature-cloud
│   └── final-cta
└── footer
```

## 2. Header

```text
header.site-header
├── brand: contrast
├── nav.main-nav
│   ├── Features
│   ├── Pricing
│   ├── Videos
│   ├── Learn
│   └── Free Tools
├── header-actions
│   ├── Book a demo
│   └── Start for free
└── menu-toggle
```

Шапка сайту відповідає за первинну навігацію. На desktop-екранах вона має горизонтальну структуру, а на мобільних екранах переходить у burger-меню.

## 3. Main

| Секція | Підкомпоненти | Призначення |
| --- | --- | --- |
| `hero` | Заголовок, опис, кнопки CTA, зображення webinar interface | Формує перше враження і презентує основну цінність Contrast. |
| `social-proof` | Аватари, заголовок, ряд логотипів | Підтверджує довіру до платформи через користувачів і компанії. |
| `ai-section` | Стрічка `new`, заголовок, опис, AI preview | Пояснює, що з одного вебінару можна отримати набір контенту за допомогою AI. |
| `engagement` | Заголовок, список функцій, preview | Демонструє інструменти залучення й утримання аудиторії. |
| `feature-cloud` | Заголовок, сітка карток функцій | Показує набір можливостей платформи у вигляді мозаїки. |
| `final-cta` | Заголовок, підпис, кнопки | Повторює основну цільову дію наприкінці сторінки. |

## 4. Footer

```text
footer.site-footer
├── footer-about
│   ├── footer-brand
│   ├── короткий опис
│   └── social-links
├── Product
├── Resources
└── More
```

Футер завершує сторінку, містить службову навігацію та повторює бренд Contrast.

## 5. Відповідність файлам

| Файл | Роль |
| --- | --- |
| `index.html` | Семантична HTML-структура сторінки. |
| `assets/css/style.css` | Desktop-стилі, адаптивність, кольори, сітки, картки та стани. |
| `assets/js/main.js` | Burger-меню і перемикання активного пункту engagement. |
| `assets/img/contrast-full.png` | Повний референс макету з пояснювальної записки. |
| `assets/img/webinar-stage.png` | Зображення hero preview. |
| `assets/img/ai-panel.png` | Зображення AI-секції. |
| `assets/img/engagement-panel.png` | Зображення engagement preview. |
| `assets/img/feature-cloud.png` | Референс мозаїки функцій. |
| `assets/img/cta-card-ref.png` | Референс фінального CTA-блоку. |

## 6. Адаптивність

| Breakpoint | Зміни |
| --- | --- |
| `max-width: 991px` | Навігація переходить у burger-меню, engagement стає одноколонковим, footer перебудовується. |
| `max-width: 768px` | Зменшуються заголовки й відступи, feature grid переходить у дві колонки. |
| `max-width: 500px` | Кнопки стають на всю ширину, логотипи клієнтів і feature cards переходять в одну або дві колонки без горизонтальної прокрутки. |

## 7. Висновок

Декомпозиція сайту Contrast показує, що сторінка складається з незалежних компонентів, які можна верстати і тестувати окремо. Такий поділ відповідає структурі, описаній у пояснювальній записці Яцківа: `Header`, `Hero`, `Social proof`, `AI section`, `Engagement`, `Feature cloud`, `Final CTA` і `Footer`.
